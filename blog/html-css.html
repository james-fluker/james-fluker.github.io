Using classes and IDs

It's important to use the right HTML elements to structure your content properly.

Classes and IDs are commonly used in web design to apply styles and functions to specific elements on the page. In this post, I'll explore the best practices associated with using classes vs IDs.

Classes group elements together when they share similar styles or functions. You can use them in HTML by adding a class attribute is added to an element - usually with a name that describes the function. You could style all of the headings on each page of a website with a specific font by adding a heading class to each heading HTML element.

You can then use your CSS to target the class and apply styles to every element with that class.

IDs are used with a specific page element. Rather than grouping many elements together they alter a specific function of an element. In the HTML you add an ID attribute to an element, and give it a unique name based on its location or function. You might want to add a button to your page that scrolled to a specific section - you could use an ID to achieve this.

You could then use JavaScript to target the ID and create a function that would scroll the user to that section.

So, how do you know when you should use classes vs. IDs? Here are some general best practices:

  1. Use classes to group elements that have similar styles or functions. That way you can can apply styles to multiple elements at once - meaning you don't have to style it in your HTML.
  2. Use IDs to target specific page elements like anchor links or JavaScript functions. Each ID should be unique and descriptive so that you know what the element is used for.
  3. Don't use IDs for styling. It's possible to target IDs in CSS, but its generally considered bad practice to do so. Use classes to style your elements as it's easier to maintain.
