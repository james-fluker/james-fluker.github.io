The basics of JavaScript

JavaScript is a programming language that makes web pages interactive. When you're using the web to engage in what seem like simple activities, there's often JavaScript code making things possible

JavaScript, HTML, and CSS

You might think that HTML is the language of the web, but usually JavaScript is referred to as the language of the web. To make it easier to understand, you might like to think of it like a house. HTML is the foundation and the structure, while CSS is the paint and decorations. In this analogy, JavaScript is the electricity - without it the house may look pretty, but nothing functions.

Control Flow and Loops

Control flow is a concept in programming that determines the order that a program runs its code. Loops are used to repeat a set of instructions until a particular set of conditions is met. Code runs in the order that it is written - from the first line in the file to the last line in the file. Structures like conditionals and loops change the flow, until the conditions have been met.

We're going to make a sandwich using control flow and loops:

  1. Check for bread - if bread is available we pass the condition and move onto the next step. If bread is unavailable the loop closes.
  2. Check for condiments - if condiments are available we pass the condition and move onto the next step. If condiments are unavailable, we won't be able to make a tasty sandwich so the loop closes.
  3. Check for toppings - we can actually break this step down more granularly - e.g., do we have Ham? If no, do we have Lunch Meat? We check for toppings, until our preferred toppings are available, or alternatives are provided. If no toppings are available we can't make a sandwich, so we close the loop.
  4. Place toppings and condiments on bread - finally, we can put our toppings (in order) and our condiments on a slice of bread. We can also include a check that the sandwich has a slice of bread added to the top.

And there we have it! The coding sandwich! Tasty!

Document Object Model (DOM)

The DOM is a programming interface. It translates the content of an HTML document into an object that programming languages like JavaScript can access and modify. HTML events are things that happen to HTML. When JavaScript is used in HTML pages, JavaScript can react on these events. Some examples of events are:

  • An HTML button was clicked
  • A form is submitted
  • The page is loaded

Most web page events are driven by non-HTML code, so virtually all web pages rely on the DOM to display and function correctly.

Accessing data from arrays and objects

Arrays and objects are ways to store data in JavaScript. Arrays store lists of items, while objects are used to store key-value pairs. Here's an example of key-value pairs using a grocery list. The key is the name of the item and the value is the quantity of items required.

Item Quantity
Apples 5
Bananas 2
Milk 1
Bread 2

Javascript code could than be used to display the value for the key on the webpage where required. A loop could be used to calculate the quantity of the key value, so that it updates dynamically. That way, every time an item from the grocery list is added to the cart, the value doesn't need to be recoded.

Functions

Functions are bits of code that perform specific tasks. Functions are useful, because they allow you to reuse code. You could write a function that sums multiple values, and then reuse that function every time you're required to add values together.
